Software::JRefactory - Third Party Classes

You can load in the class descriptions for JDK1.2 and other applications that provide you with the source code. By transforming these java files into a stub, they won't be included in your metrics and you won't be able to perform any refactorings on them. However, they do provide a way to visualize how your classes relate to the JDK.

I've used this mechanism to view the relationship between my code and the JDK 1.2 and IBM's XML4J.

Learn more about the stub generation tool.
